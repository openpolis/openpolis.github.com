Executing Django views

 Mon 01 July 2013 by guglielmo
Filed under howto Tags django python

The code within django views is executed during the HTTP request-response workflow and there is little chance of executing it outside that context.

There can be, however situations in which executing it is exactly what you want. For example if you need to test the views efficiently, or you want to generate context caches through management tasks.

Starting from http://tech.novapost.fr/django-unit-test-your-views-en.html, where a Django’s test client replacement is described, I’ve implemented a management task to execute the get_context_data method of a class-based view. This allowed me to easily debug the code, understanding some of the quirks in it, and to use it more generally, as a tool to manually (re-)generate the related context cache, without using a headless browser.

At the core of the technique, is the following function, that receives a CBV instance, along with:

  • the request,
  • an object instance (since we’re working with a DetailView class),
  • the same args and kwargs that’d be used in reverse().
def setup_view(view, request, object, *args, **kwargs):
    """Mimic as_view() returned callable, but returns view instance.

    args and kwargs are the same you would pass to ``reverse()``

    """
    view.request = request
    view.object = object
    view.args = args
    view.kwargs = kwargs
    return view

Having this function handy in some package, allows you to write the following piece of code:

from myutils_package import setup_view

view = setup_view(
    MyView(),
    RequestFactory().get("/mypath/{0}".format(slug)),
    obj_instance
)

context = view.get_context_data()

that will compute the view’s context, as if a client had visited the url, with a get request.

If the class definition contains a cache decorator, then the cache is stored:

class MyView(DetailView):
    model = MyModel
    context_object_name = 'mymodel'

    @cached_context
    def get_context_data(self, **kwargs):

        context = super(MyModelView, self).get_context_data(**kwargs)
        ...
        return context

The decorator definition:

def cached_context(get_context_data):
    """
    This decorator is used to cache the ``get_context_data()`` method
    called by a ``get()`` or ``post()`` in the views.
    It generates a unique key for the request,
    checks if the key is in the cache:
    if it is, then it returns it,
    else it will generate and save the key, before returning it.
    """

    def decorator(self, **kwargs):
        key = 'context' + self.request.get_full_path()
        context = cache.get(key)
        if context is None:
            context = get_context_data(self, **kwargs)
            cache.set(key, context)
        return context
    return decorator
